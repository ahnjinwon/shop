# 비대칭 암호화 (Asymmetric Encryption)

## 개념

- **키 쌍 (public/private key)**: 공개 키는 외부에 공유하고, 개인 키는 비밀로 유지한다.
- **암호화/복호화**:
  - 공개 키로 암호화한 데이터는 개인 키로만 복호화할 수 있다.
  - 반대로, 개인 키로 서명한 데이터는 공개 키로 검증할 수 있다.
- **용도**:
  - 기밀성: 수신자의 공개 키로 암호화하면 수신자의 개인 키를 가진 사람만 읽을 수 있다.
  - 무결성/인증(전자서명): 송신자가 개인 키로 서명하고, 수신자는 공개 키로 서명을 검증한다.

## 장점과 단점

- **키 배포가 편리**: 공개 키는 누구에게나 주어도 되므로, 대칭키처럼 비밀리에 전달할 필요가 없다.
- **개인 키만 지키면 안전**: 개인 키가 유출되지 않는 한 데이터 기밀성과 서명 검증이 안전하다.
- **다만 속도는 느리다**: 대칭키 암호화보다 계산량이 많아 속도가 느리므로, 실제 데이터는 보통 대칭키로 암호화하고 키 교환/서명에만 비대칭 암호를 사용한다.

## 일반적인 사용 패턴

1. **키 교환**: 비대칭 암호로 세션 키(대칭 키)를 안전하게 교환한 뒤, 실제 데이터 전송은 빠른 대칭 암호로 암호화한다(TLS 등).
2. **서명/검증**: 데이터는 대칭키로 암호화하고, 해시를 개인 키로 서명하여 무결성을 보장한다.
3. **인증서**: 공개 키에 대한 신뢰를 보장하기 위해 CA가 발급한 인증서(예: X.509)를 사용한다.

## 대표 알고리즘

- RSA, DSA, ECDSA, Diffie–Hellman(ECDH) 등
- 선택 시 키 길이, 성능, 표준 지원 여부를 고려한다.
